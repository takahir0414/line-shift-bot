# シフト管理AI - LINE Bot（希望回収）

将泰庵グループ シフト管理AIの第1フェーズ。LINEを起点に、スタッフの「初回登録（氏名・所属店舗・雇用形態）」と
「2週間単位のシフト希望提出（出勤希望日・開始/終了時刻・社員のみ休み希望）」を回収するBotです。

## 会話フロー

1. **初回登録**：友だち追加 → 氏名（テキスト入力） → 所属店舗（ボタン） → 雇用形態（ボタン：社員／アルバイト・パート）
2. **希望提出**：「希望を出す」と送信 → 2週間分（14日）の出勤希望日をボタンでタップ（出勤希望→休み希望(社員のみ)→未選択の順に循環） → 「次へ」 → 選択した出勤希望日ごとに開始・終了時刻をLINE標準の時刻ピッカーで入力 → サマリー確認 → 「この内容で提出」

## ディレクトリ構成

```
api/webhook.js     ... LINE Webhookのエントリポイント（署名検証・イベントルーティング）
lib/constants.js   ... 店舗一覧・雇用形態などの設定値
lib/session.js      ... スタッフごとの会話状態の保持（現状メモリ実装。本番はRedis等に差し替え必須）
lib/flex.js          ... Flex Message / 時刻ピッカーのビルダー
```

## セットアップ

1. LINE Developersコンソールで Messaging API チャネルを作成し、
   `チャネルアクセストークン` と `チャネルシークレット` を取得する。
2. `.env.example` を `.env` にコピーし、上記の値を設定する。
3. 依存パッケージをインストール：
   ```bash
   npm install
   ```
4. Vercelにデプロイ：
   ```bash
   vercel deploy
   ```
   デプロイ後、環境変数 `LINE_CHANNEL_ACCESS_TOKEN` / `LINE_CHANNEL_SECRET` をVercelのプロジェクト設定に登録する。
5. LINE DevelopersコンソールのWebhook URLに `https://<your-vercel-domain>/api/webhook` を設定し、Webhookを有効化する。
6. LINE公式アカウントの「応答メッセージ」はOFF、「Webhook」はONにしておく（Bot側の自動応答と競合させないため）。

## 重要な制約・引き継ぎ事項（次フェーズ向け）

- **セッション保存先**：`lib/session.js` は検証用にメモリ実装（`Map`）です。Vercelのサーバーレス関数は
  インスタンスが使い回されないため、本番では Upstash Redis（Vercel Marketplaceから連携可）や Vercel KV
  など永続ストアに**必ず**差し替えてください。差し替え方法はファイル冒頭のコメントに記載しています。
- **希望データの送信先**：`api/webhook.js` の `action=submit` ハンドラ内に
  `TODO（次フェーズ）` として明記している通り、現状は `console.log` で確認できるのみです。
  要件定義書の「②分類・過不足アラート機能」「データ要件：希望シフト」に従い、ここでDBへの保存・
  処理基盤への連携を実装してください。
- **店舗一覧**：`lib/constants.js` の `STORES` は店舗名で仮置きしています。要件定義書「9. 今後の検討事項」の
  店舗名・店舗コード確定後に更新してください。
- **返信方式**：イベント処理を非同期化している都合上、`replyToken`（1回限り・有効期限あり）ではなく
  `pushMessage`（友だち1人あたり月あたりの無料送信数に上限あり）を使用しています。送信数が課題になる場合は
  replyTokenベースの同期処理への変更を検討してください。
- **氏名の入力**：要件として「本名で入れたい」とのことなので自由テキストのままにしていますが、
  表記ゆれ（全角/半角スペース、旧字体等）の正規化や、提出後の本部側での名寄せチェックが必要になる可能性があります。
